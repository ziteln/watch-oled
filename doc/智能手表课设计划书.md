### **五、项目风险与应对预案（新增物料物流、代码仓库相关风险）**

1.  物料物流延期风险：OLED、传感器等核心物料延迟送达 → 先用实验室备用模块完成软件代码开发，物料到货后直接替换硬件调试；每日同步物流记录，及时补充急缺器件。
2.  硬件电路故障：I2C 通信异常、供电不稳 → 成员 A 分模块分步接线调试，面包板预留独立测试分区，单独测试电源、传感、显示模块。
3.  FreeRTOS 多任务冲突、死锁：成员 B 使用互斥信号量保护 I2C 共享总线，单任务单独调试通过后再进行多任务联调，代码分版本提交 GitHub 方便回滚排查。
4.  GitHub 仓库协作风险：代码 / 文档覆盖丢失 → 每日固定时间提交、备注清晰，每周完整备份仓库，保留历史版本。
5.  扩展功能调试困难（蓝牙 / 计步）：优先保障基础要求（时间、传感器、OLED 双页面）全部完成，再攻坚扩展项，保底完成至少 1 项扩展功能保证中等及以上成绩。
